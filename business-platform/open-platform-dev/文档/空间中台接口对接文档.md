# 空间资源管理服务接口对接文档

## 1. 概述

本文档描述了空间资源管理服务的所有API接口，供开发人员对接使用。

### 1.1 服务基础信息

- **服务名称**: hkt-blade-space-resource-service
- **接口版本**: v1
- **基础路径**: `/v1/space`
- **接口协议**: RESTful API
- **数据格式**: JSON

### 1.2 认证方式

所有接口需要在请求头中携带认证Token：

```
Authorization: Bearer {token}
```

### 1.3 通用响应格式

所有接口统一返回以下格式：

```json
{
    "code": 200,           // 状态码：200成功，其他为失败
    "success": true,       // 是否成功
    "data": {},            // 返回数据
    "msg": "操作成功"       // 提示信息
}
```

### 1.4 分页响应格式

分页接口返回格式：

```json
{
    "code": 200,
    "success": true,
    "data": {
        "records": [],       // 数据列表
        "total": 100,        // 总记录数
        "size": 10,          // 每页大小
        "current": 1,        // 当前页码
        "pages": 10          // 总页数
    },
    "msg": "操作成功"
}
```

## 2. 空间节点管理接口

**Controller**: SpaceNodeController  
**基础路径**: `/v1/space/node`

### 2.1 新增空间节点

**接口描述**: 创建一个新的空间节点

| 项目 | 说明 |
|------|------|
| URL | `POST /v1/space/node/create` |
| Content-Type | application/json |
| 认证 | 需要登录 |

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | String | 是 | 空间名称 |
| area | BigDecimal | 否 | 面积 |
| parentId | String | 否 | 父节点id（新增根节点时为空） |
| rootId | String | 否 | 根节点id（新增根节点时为空） |
| levelId | String | 是 | 层级id |
| path | String | 否 | 物化路径，用于跨层级导航搜索 |

**请求示例**:
```json
{
    "name": "A栋办公楼",
    "area": 5000.00,
    "parentId": null,
    "rootId": null,
    "levelId": "level_001",
    "path": null
}
```

**响应参数**:

| 参数名 | 类型 | 说明 |
|--------|------|------|
| data | String | 新创建的节点id |

**响应示例**:
```json
{
    "code": 200,
    "success": true,
    "data": "node_123456",
    "msg": "操作成功"
}
```
