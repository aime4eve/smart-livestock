设计改进说明

痛点 1：操作杂乱 → 清晰有序
  • 移除硬编码 P0 假数据，全部来自真实 API
  • 状态分段控件 + 类型 chips 均可点击实时过滤
  • 每张卡片独立操作（点击看详情 → 操作按钮）
  • 长按进入批量选择，支持批量已读/忽略
痛点 2：浏览不丝滑 → 流畅自然
  • 告警按日期分组（今天/昨天/更早）
  • 已读/未读通过色条 + 绿点视觉区分
  • 摘要条点击快速跳转到严重/警告/待处理
  • 已处理视图与活跃视图风格统一但视觉降级
痛点 3：看不到细节 → 信息完整
  • 卡片展示：严重程度、类型图标、牲畜编号、围栏、时间、来源
  • 详情面板：完整元数据网格 + 处置时间线
  • 「地图定位」按钮从详情直接跳转到地图上的告警位置
  • 围栏告警展示偏离距离，健康告警展示体温值
与牧场页的联动
  • 牧场页地图上用色标标记告警牲畜位置
  • 顶部告警横幅显示严重告警数，点击进入告警中心
  • 底部看板卡片按类别显示告警数，点击筛选
现有问题 告警页是独立的 SingleChildScrollView，与牧场页无联动；P0 告警行是硬编码假数据；状态/类型 chips 是装饰性的不可点击；操作按钮只作用于第一条告警；没有详情页、没有批量操作、没有已读状态、没有日期分组
重设计后 告警从牧场页自然延伸（横幅→列表→详情→地图定位）；完整的数据驱动列表；功能性筛选；每卡片独立操作 + 批量模式；富信息详情面板含处置时间线；已读/未读视觉区分；按日期分组浏览
