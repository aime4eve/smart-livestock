📍 真值基准管理 ⑥ 完全不变

RTK 真值点（33 个）与计划路线管理维持现状。轨迹动态检验的真值来自导入的 RTK 轨迹文件（随会话存储），不占用此处的固定真值点与路线，两种动态检验方式并存：

· 路线动态（现有）：真值 = 固定 RTK 点序列，适合反复使用的标准巡回路线
· 轨迹动态（新增）：真值 = 随行 RTK 轨迹，适合任意路径的外场检验

🧭 轨迹动态检验 × 现有界面关系总览

GPS 质量检验页（/admin/gps-quality）
  • Tab 1 检验列表变更 左栏工具条 +🛰 导入入口（①）｜设备分组行 +「轨迹」类型标记（②）
    • 新建检验弹窗不变仍只创建 静态（设备+真值点）/ 路线动态（设备+路线）；轨迹检验只由文件导入产生
    • 批量导入会话弹窗不变现有 Excel 模板批量建静态/路线动态会话，与轨迹导入是两条独立入口
    • 🛰 导入 RTK 轨迹数据向导新增3 步：上传文件 → 校验与配对预览（±60s 容差可调）→ 按设备生成「轨迹动态」会话
    • 检验时间轴变更新增紫色「轨」段（③），点击切换报告
    • 报告区按类型分派（④）扩展
      • 静态 → 散布图 + 抖动分析不变
      • 路线动态 → 路线匹配图 + 覆盖率不变
      • 轨迹动态 → 配对报告新增配对概览 / 轨迹对比图 / 误差分布 / 逐样本明细（文件 vs 库内来源）/ 未配对清单 / 静动态对比
  • Tab 2 真值基准不变（⑥）33 个 RTK 点 + 计划路线；轨迹真值随导入文件存储，不占用固定真值点
  • Tab 3 质量对比数据源扩展（⑤）「动态」一侧 = 路线动态 + 轨迹动态，逐设备标注来源；对比逻辑不变
📦 数据归属：轨迹动态会话 = 设备 + 时间窗（由文件采集时间 min/max 推导）+ 随行 RTK 轨迹点集合；与静态/路线动态共用同一张 gps_quality_tests 会话表（新增 test_type=TRAJECTORY），轨迹点存新表。
