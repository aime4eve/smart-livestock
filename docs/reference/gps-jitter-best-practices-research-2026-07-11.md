## 一、信号级滤波方法

### 1.1 滑动平均（Moving Average）

对最近 N 个位置样本取平均，平滑高频抖动。计算成本最低，适合嵌入式设备和实时管道。

- **适用场景**: 采样频率较高（秒级到分钟级）、噪声为白噪声的情况
- **在我们场景的问题**: 30 分钟采样间隔下，窗口内样本跨越时间太长（N=3 = 1.5 小时），会抹平真实的缓慢移动

### 1.2 中值滤波（Median Filter）

对最近 N 个点取中值（纬度、经度分别取 median），擅长去除孤立"野点"（单次跳变 100m+）而不模糊真实移动边界。

- **优势**: 对离群点鲁棒，不会像均值滤波那样被极端值拉偏
- **适用**: 作为预处理步骤，去除明显异常点

### 1.3 Kalman 滤波

通过预测-更新循环建模系统状态（位置、速度），自然分离测量噪声和真实运动。IEEE 研究（2019）证明 Kalman 滤波在 geofencing 场景下可将 GPS 误差从 ~15m 显著降低。

- **局限（我们的场景）**: 30 分钟采样间隔使预测步过长，预测-修正效果有限。原文判断正确——等上报频率提升或设备提供 HDOP/accuracy 字段后再考虑。
- **增强方向**: 融合加速度计/IMU 数据后 Kalman 效果显著提升。如果后续设备上报 accelerometer 数据（我们已有加速度计数据），可重新评估。

### 1.4 Savitzky-Golay 滤波

通过局部多项式拟合平滑数据，保留信号的高阶矩（峰值、宽度），比滑动平均更好地保留真实运动特征。

- **适用**: 事后分析/离线处理，计算成本高于滑动平均和 Kalman
- **来源**: [IEEE xplore](https://xplorestaging.ieee.org/ielx7/6287639/10005208/10075552.pdf)

### 1.5 速度/物理可行性过滤

丢弃隐含物理不可能移动的定位点：如果两点间隐含速度超过可配置阈值（如行人 >5 m/s，牛 >3 m/s），标记为异常点，丢弃或降权。

- **优势**: 逻辑简单，无需调参
- **适用**: 任何采样频率下都有效，可作为第一道防线

## 二、围栏/Geofence 防误报技术

### 2.1 缓冲区域（Buffer Zone）

在围栏边界外扩展一个缓冲区吸收 GPS 误差。这是最广泛引用的技术。

| 类型 | 描述 |
|------|------|
| **静态缓冲** | 围栏多边形向外扩展固定距离（通常 15-50m，取决于 GPS 精度） |
| **弹性/动态缓冲** | 首次合法进入后临时扩大围栏，使后续抖动不会触发误退出；设备明确离开后恢复原始大小（US Patent 6,496,775） |
| **过渡区域** | 定义嵌套两层：外层过渡区 + 内层核心区。进入外层触发"待确认"，进入内层立即确认（US Patent App. 2024/0177102） |

**畜牧业虚拟围栏的实际数据**：

- 商用 NoFence© 项圈：静态测试精度误差 3.3m ± 2.5 SD，精度误差 4.6m ± 3.2 SD
- 自由放牧牛只身上（移动测试）：误差翻倍 → 10.8m ± 6.8 SD（动物身体遮挡 + 地形）
- Vence™ 项圈静止测试：平均水平误差 37 ft (~11.3m)，极端离群值达 9,195 ft
- **建议缓冲**: 畜牧业推荐 100 ft (~30m) 缓冲，避免在崎岖地形/密林设置虚拟围栏边界

> 来源: [Animal Biotelemetry, 2024](https://link.springer.com/article/10.1186/s40317-024-00389-8), [UA Extension, 2024](https://extension.arizona.edu/sites/extension.arizona.edu/files/pubs/az2087-2024.pdf)

### 2.2 滞留时间/防抖（Dwell Time / Debounce）

要求设备在围栏内（或外）停留**最短时间**后才触发告警：

- **典型值**: 30 秒到 5 分钟，取决于场景
- **Android**: 可通过 `setLoiteringDelay()` 配置
- **物流场景**: 2-5 分钟确认车辆真正停靠而非路过或瞬时漂移

### 2.3 连续确认（Vote of Two）

必须**连续两个（或更多）GPS 定位点**一致地显示新状态，才确认围栏穿越。如果点 #1 显示退出但点 #2 仍显示在内，抑制告警。

> 来源: US Patent 8,125,332

### 2.4 精度圆/误差椭圆感知

将 GPS 接收器报告的**水平精度/误差椭圆**纳入判断：

- 仅当**整个精度圆**都在围栏内时触发进入事件
- 仅当**整个精度圆**都在围栏外时触发退出事件
- 防止点坐标跨过边界但真实位置（在误差半径内）可能在原始侧的边缘抖动

### 2.5 冷却期（Cooldown）

围栏事件触发后，强制执行**冷却时间**（如 2-5 分钟），期间同一围栏不再生成事件。直接抑制 GPS 抖动在边界处的快速 flip-flop。

### 2.6 行进路径分析（Path-of-Travel）

不反应单次越界，而是分析连续点的**轨迹**：

- 设备**持续远离**围栏（多点一致）→ 真实退出
- 设备 zigzag 或弹回 → 可能是抖动，抑制告警

> 来源: US Patent 6,496,775

### 2.7 三区虚拟围栏算法 + 方向门控

韩国研究论文提出的分层方案：

| 区域 | 用途 | 刺激 |
|------|------|------|
| 安全区 | 正常放牧 | 无 |
| 警告区 | 缓冲区，动物接近边界 | 弱刺激鼓励返回 |
| 危险区 | 接近最终边界 | 强刺激防止逃逸 |

结合**方向门控**（heading filter）：仅当动物**朝向排除区移动**时触发刺激；动物转身或改变方向 ≥45° 立即停止。

> 来源: [KoreaScience](http://koreascience.or.kr/article/CFKO201731342442501.pdf), US Patent 9,107,395

## 三、停留点检测（Stay Point Detection）

| 技术 | 描述 |
|------|------|
| **距离 + 时间阈值** | 设备在距离阈值 D（如 30m，匹配 GPS 误差半径）内停留超过时间 T（如 2-5 分钟）→ 停留点。天然容忍 GPS 抖动 |
| **DBSCAN 密度聚类** | 按空间邻近度聚类连续 GPS 点，超过最小时长的聚类为停留点。对离群点鲁棒，适合离线/事后分析 |
| **速度检测** | 滤波后速度（Kalman 或滑动平均）降到阈值以下（如 <1 m/s）持续最小时长 → 判定静止。速度视角通常比原始位置数据更干净 |

**我们对口的场景**：30 分钟采样间隔下，距离+时间阈值法最实用。D 设为抖动直径的 1.2-1.5 倍（~15m），T 设为 2-3 个采样周期（1-1.5 小时）。

## 四、畜牧场景特定研究

### 4.1 低采样频率的可行性

- **30 分钟间隔**对精细行为分类（短暂进食 vs 休息）有困难，但对**日活动预算**（总放牧时长/天、移动距离估计）和**栖息地偏好分析**仍可行
- 牛是大型慢速动物，大部分时间执行少量行为（吃草、反刍、休息），即使 300 秒间隔分类精度仍相对较高
- **推荐方案**: 30 分钟 GPS + 连续/高频加速度计数据，在保持电池寿命的同时维护行为分类精度
- **加速计可用**: USDS ARD 研究发现，GPS 间隔 >2 分钟时，最佳策略是用加速计指标预测行为（30s epoch），再汇总到每个 GPS fix 间隔

> 来源: [USDA ARS, 2024](https://www.ars.usda.gov/ARSUserFiles/30123025/Publications/2024/Cunningham%20et%20al%202024_Smart%20Ag%20Tech_Optimal%20device%20for%20cattle%20activity%20budgets.pdf), [PubMed 38791723](https://pubmed.ncbi.nlm.nih.gov/38791723/)

### 4.2 畜牧轨迹聚类方法

| 方法 | 描述 |
|------|------|
| **隐马尔可夫模型（HMM）** | 基于方向速度和角速度识别停留区域和行为状态（吃草、休息、移动）。可处理不同 GPS 间隔 |
| **凝聚层次聚类** | 用轨迹对之间的时间平均距离聚类，每簇建模为高斯源用于长期运动预测 |
| **空间热点分析（Getis-Ord Gi\*）** | 1m×1m 网格识别行为 GPS 点的统计显著聚类（吃草热点 vs 冷点） |

> 来源: Guo et al. 2009, *Ecological Modelling*; [Sci. Agric.](https://qa1.scielo.br/j/sa/a/h5BmKwWf9FX6QfpXzNtt7rS/?format=pdf&lang=en)

## 五、轨迹简化算法

### 5.1 Douglas-Peucker

经典线简化算法，递归去除对整体形状贡献最小的点。保留轨迹的宏观形状，丢弃冗余顶点。

- **优势**: 计算简单，效果好，是最广泛使用的轨迹简化算法
- **可直接应用**: 对轨迹展示做后端/前端简化，只保留方向变化显著的点

### 5.2 滑动窗口 + 密度骨架化

EGC 2019 提出的三阶段框架：

1. **重采样 + 高斯模糊**: 构建密度面，消除噪声
2. **轨迹调整（Slide 工具）**: 将噪声点推向高密度走廊（结合表面梯度、距离均匀性、角度、动量四个向量）
3. **骨架提取 + Douglas-Peucker 简化**: 提取路网图

> 来源: [EGC 2019](https://imae.udg.edu/egc2019/doc/BookAbstractsEGC2019.pdf#26)

## 六、业界推荐的完整管道架构

```
原始 GPS 数据流
    │
    ├─► [速度/物理可行性过滤] ── 丢弃不可能的定位点
    ├─► [中值/均值滤波] ── 平滑位置
    ├─► [加速度计交叉验证] ── 静止时抑制漂移
    ├─► [停留点检测器] ── 分类静止 vs 移动
    └─► [围栏引擎]
           ├─ 精度圆感知边界检查
           ├─ 连续确认（Vote of Two）
           ├─ 滞留时间防抖
           └─ 冷却期强制
                │
                └─► 告警分发（频率上限、静默时段、行为上下文）
```

## 七、对我们的建议总结

结合 [审阅意见](./gps-jitter-analysis-review-2026-07-11.md) 和本次调研，针对**30 分钟采样间隔 + 消费级 GPS + 畜牧场景**的推荐优先级：

### 第一优先级：围栏防误报（影响最大）

| 措施 | 复杂度 | 效果 |
|------|--------|------|
| 连续确认（Vote of Two） | 低 | 消除单点抖动误报 |
| 冷却期（同围栏 N 分钟内不重复触发） | 低 | 阻断快速 flip-flop |
| 距离阈值过滤（位移 < 抖动直径 1.2× = 跳过围栏检测） | 低 | 静止牛不产生告警 |

### 第二优先级：轨迹展示

| 措施 | 复杂度 | 效果 |
|------|--------|------|
| 前端位移阈值过滤（两点 < ~12m → 合并为同一停留点） | 低 | 轨迹不再呈锯齿线 |
| Douglas-Peucker 简化（后端返回简化轨迹） | 中 | 同时减少数据传输量 |

### 第三优先级：统计修正

| 措施 | 复杂度 | 效果 |
|------|--------|------|
| 停留点聚类 + 移动段分段计算距离 | 中 | 不过度统计抖动、不低估真实移动 |
| 速度过滤（异常点剔除） | 低 | 排除明显错误数据 |

### 待条件成熟后评估

| 措施 | 前提条件 |
|------|----------|
| Kalman 滤波 | 上报频率提升至分钟级 或 设备提供 HDOP/accuracy |
| 加速度计交叉验证 | 加速计数据与 GPS 数据时间对齐（目前三维不齐，见 [AI-Platform seed 问题](../memory/../)) |
| 精度圆感知边界 | 设备上报 accuracy/hdop 字段 |
