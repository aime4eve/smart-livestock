## 4. 推荐的治理策略

基于以上分析，建议对 blade 平台 GPS 数据采用**三层过滤管道**：

### Layer 1: 速度门控（Speed Gate）
- 过滤物理不可能的速度（如牛 > 3 m/s 的瞬间位移）
- 使用相邻点间的 haversine 距离 / 时间间隔计算速度
- 过滤掉极端野点（如突然跳到 200m 外再跳回来）

### Layer 2: 滑动窗口中值滤波（Median Filter）
- 窗口大小 3-5 个点（1.5-2.5 小时）
- 对经度和纬度分别取中值
- 有效消除孤立野点，不模糊真实移动边界

### Layer 3: EMA 稳定位置（Stable Position）
- 阈值基于设备 P90 抖动距离设定（建议 15-20m）
- 只有位移超过阈值才触发围栏检测
- 稳定位置用 EMA 微调，平滑小幅度漂移

```
  blade平台GPS → [速度门控] → [中值滤波] → [EMA稳定位置] → 围栏检测
                    ↓ 野点丢弃       ↓ 去孤点        ↓ 去抖动
```

## 结论

1. **blade 平台原始数据比 gps_logs 抖动更大**（80-335m vs 11m），
   可能存在精度截断差异或上报间隔不均匀
2. **中值距 RTK 仅 3-9m**，说明大部分数据质量可用，问题在于野点
3. **三层过滤策略**可以有效治理：速度门控去野点 → 中值滤波平滑 → EMA 稳定位置用于围栏
4. **推荐阈值**: 滑动窗口 3-5 个点，EMA 阈值 15-20m
