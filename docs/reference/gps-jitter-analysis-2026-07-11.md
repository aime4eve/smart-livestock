# GPS 坐标抖动问题分析与建议

- **日期**: 2026-07-11
- **设备**: HKT-11-01（dev_eui `00956906000285d8`，device_id=104）
- **牲畜**: livestock_id=66（livestock_code `HKT11`，绑定 farm_id=1）
- **场景**: 设备静止放置于 11 号点位（一期楼顶），未移动

## 问题描述

11 号点位是用 RTK 设备实际测量出的精确 GPS 坐标（28°14′47.73855″N, 112°51′05.79753″E）。HKT-11-01 设备一直放置在该点位没有移动，但从实际采集的轨迹数据来看，坐标每次都不同，存在明显的"漂移"现象。这是消费级 GPS 芯片的固有限制（精度 3-10m），需要在本系统中妥善处理。

### 分析结论

1. **典型消费级 GPS 表现**: 精度 3-10m，存在约 2m 的系统性偏差，静止时在 10m 直径范围内抖动。
2. **非随机噪声**: 13 个离散点位反复出现，说明 GPS 芯片在卫星几何变化时输出有限几个"锁定位置"，不是均匀随机分布。
3. **系统性偏差小**: 平均中心距 RTK 真值仅 2.2m，芯片本身精度可以接受。

## 对系统的三层影响

### 1. 轨迹展示

一头牛在栏杆上不动，地图上却画出一条 10m 范围的锯齿线，看起来像在来回走动。用户看到这种轨迹会对系统失去信任。

### 2. 围栏告警

围栏越界检测依赖 GPS 坐标和围栏几何的距离计算。如果围栏边界恰好在抖动范围内（比如 11 号点位离围栏边界 8-12m），静止的牛就会反复触发 FENCE_APPROACH / FENCE_BREACH 告警。dev 环境日志中已观察到 livestock 65 的 FENCE_APPROACH -> FENCE_BREACH 自动升级循环，很可能就是这个原因。

### 3. 统计距离

轨迹距离按每两个点的 haversine 距离累加，536 个抖动点每个差 5-10m，累加出来的"移动距离"可能凭空多出几百米。

## 建议方案

### 第一层：前端轨迹展示（快速见效）

在 `trajectory_sheet.dart` 的轨迹渲染中增加最小位移阈值过滤：连续两点之间距离小于阈值（如 15m）的，合并为同一个停留点。静止的牛在地图上只显示一个标记或小范围半透明区域，而不是锯齿线。

- 只影响展示，不碰后端数据
- 改动最小，效果立竿见影

### 第二层：后端引入"停留点"概念（中期）

在展示和统计层面引入"停留点聚类"逻辑：连续 N 个点（如 3 个以上，约 1.5 小时）都在阈值范围内，归为一个停留点。

- 轨迹由"停留点 + 移动段"组成，不是原始点序列
- 距离统计只累加停留点之间的位移
- 围栏检测基于停留点而不是原始点，减少误报
- 原始 GPS 数据仍保留，用于审计和调试

### 不建议的方案

不建议现阶段上 Kalman 滤波。30 分钟一次的上报间隔太稀疏，Kalman 的预测-修正模型在这个频率下效果有限，反而增加复杂度。等上报频率提升到分钟级、或设备开始提供 HDOP/accuracy 字段时再考虑。
