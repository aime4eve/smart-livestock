# B2B Admin 用户体验重新设计方案

## 背景

B2B Admin（B端管理员）当前有 4 个页面：概览看板、牧场管理、合同信息、对账。存在以下问题：

1. 概览看板和牧场页有大量重复（都展示牧场列表）
2. NavigationRail 无分组，运营功能和商务功能混在一起
3. 概览页承载过多内容但缺乏聚焦点
4. 合同/对账是静态信息，交互单一
5. 移动端适配不够友好

## 设计决定

| 决定项 | 选择 | 理由 |
|--------|------|------|
| 核心定位 | 业务运营全景 | 牧场运营+合同状态+收入对账三者并重 |
| 布局结构 | 分组侧边栏 | 运营区（概览+牧场）和商务区（合同+对账）分组，心理模型清晰 |
| 概览页内容 | 标签页分区 | KPI看板/牧场概览/告警动态三个tab，手机端友好 |
| 侧边栏项目 | 4项保留 | 概览/牧场/合同/对账，但分组展示 |
| 退出登录 | 侧边栏底部 | 已实现，保持不变 |

## 用户旅程

### 旅程1：日常巡检（高频，每天1-2次）
```
登录 → 自动进入概览页
  → 扫一眼 KPI 指标（牧场数/牲畜/牧工/告警）
  → 扫一眼收入 Hero 卡片
  → 有告警？切换到"告警动态" tab 查看详情
  → 需要处理？点击告警跳转
```

### 旅程2：牧场管理（中频，每周几次）
```
侧边栏点击"牧场管理"
  → 查看牧场列表（汇总栏：总牧场/总牲畜/总牧工/总设备/告警数）
  → 新建牧场 → 填写表单（名称/负责人/经纬度/面积）
  → 点击牧场卡片 → 进入牧场详情
    → Tab "地图与围栏"：地图+围栏+离线瓦片
    → Tab "牧工"：牧工列表+分配牧工
  → 操作菜单：变更负责人/编辑名称
```

### 旅程3：商务查看（低频，每月/每季度）
```
侧边栏"合同信息" → 查看合同状态/有效期/分润比例
侧边栏"对账" → 查看分润周期列表 → 点击查看明细
```

## 页面设计

### 1. 侧边栏（NavigationRail → 分组式）

**Flutter 实现：NavigationRail 替换为自定义侧边栏**

```
┌─────────────────┐
│  B端控制台        │  ← 标题
│                  │
│  ── 运营管理 ──   │  ← 分组标签（灰色小字）
│  📊 概览          │  ← 选中态高亮
│  🐄 牧场管理      │
│                  │
│  ── 商务管理 ──   │
│  📄 合同信息      │
│  💰 对账          │
│                  │
│                  │
│  👤 管理员        │  ← 底部用户信息
│  🚪 退出登录      │
└─────────────────┘
```

**改动文件**: `demo_shell.dart` 的 `_B2bAdminShell`

将 NavigationRail 替换为自定义侧边栏（Container + Column + InkWell），支持：
- 分组标签
- 图标+文字的菜单项
- 底部退出按钮（已有）
- 宽度固定 200px（桌面端）/ 自动折叠为图标模式（窄屏）

### 3. 牧场管理页（简化）

**改动文件**: `b2b_farm_list_page.dart`

当前牧场管理页展示汇总条 + 牧场卡片列表，与概览页 Tab 2 内容高度重复。

**处理方式**：牧场管理页保留但重新定位为"高级管理入口"：
- 保留汇总条
- 保留牧场卡片列表（含 PopupMenu 操作菜单）
- 新建牧场对话框
- 概览页 Tab 2 是简化版（只有卡片，无菜单操作）

或者更简洁的方案：**牧场管理页重定向到概览页 Tab 2**，侧边栏"牧场管理"点击后跳转到概览页并自动切换到 Tab 2。这样零重复代码。

### 4. 合同信息页（保持不变）

`b2b_contract_page.dart` 已有完整设计，保持不变。

### 5. 对账页（保持不变）

`b2b_revenue_page.dart` 已有完整设计，保持不变。

## 文件改动清单

| 文件 | 改动类型 | 说明 |
|------|---------|------|
| `demo_shell.dart` | 重构 | `_B2bAdminShell` 从 NavigationRail 改为自定义分组侧边栏 |
| `b2b_dashboard_page.dart` | 重构 | 从单页滚动改为 TabBarView（KPI看板/牧场概览/告警动态） |
| `b2b_farm_list_page.dart` | 简化 | 保留或合并到概览页 Tab 2 |
| `b2b_worker_detail_page.dart` | 不变 | 牧场详情页（地图+围栏/牧工） |

## 假设

- 分组侧边栏在 Web 端宽度 200px，手机端折叠为底部导航或抽屉
- 概览页 3 个 Tab 的数据全部来自 `/b2b/dashboard` API，无新后端需求
- 告警动态 Tab 的详细告警列表需要后端增强（当前只返回摘要），可先用摘要数据
- 牧场管理侧边栏项可以重定向到概览页 Tab 2，避免代码重复
