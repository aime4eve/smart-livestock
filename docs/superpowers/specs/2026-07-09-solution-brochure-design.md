## 2. 页面结构（8 区块叙事）

### ① Hero — 价值主张

- **背景色**：`#2F6B3B` 全宽
- **内容**：大标题「智慧畜牧，数据驱动的精准养殖」+ 副标题 + 4 个核心数字卡片
- **数字**（客户价值视角）：
  1. **3 年** 设备续航 — 一次安装，长期免维护
  2. **72 小时** 提前预警 — AI 在临床症状出现前发现异常
  3. **5–10 公里** LoRaWAN 覆盖半径 — 偏远牧场无需建基站
  4. **14 天** 全功能免费试用 — 零风险上手
- **CTA**：白色边框按钮「预约演示」+ 填充按钮「免费试用」
- **动画**：标题淡入上移

### ② 痛点 — 行业挑战

- **背景色**：`#F8F6F0`
- **内容**：区块标题「传统牧场面临的挑战」+ 4 列痛点卡片
- **痛点**：
  1. 人工巡栏效率低 — 500 头牧场每天巡栏 >3 小时
  2. 疾病发现不及时 — 依赖经验，早期症状难察觉
  3. 牲畜走失盗窃 — 散养模式下位置不可见，出事后追溯困难
  4. 网络覆盖差 — 偏远牧场蜂窝信号不稳定，传统数字化工具体验中断
- **动画**：卡片滚动渐现（IntersectionObserver）

### ③ 方案 — 三位一体

- **背景色**：白 `#FFFFFF`
- **内容**：区块标题 + 三列流程（IoT 感知层 → AI 分析引擎 → 云平台管理），列间箭头连接
- **布局**：3 列，每列包含图标、标题、3 个要点
- **动画**：无

### ④ 场景 — 六大核心能力

- **背景色**：`#E3F0E4` 浅绿底
- **内容**：3×2 卡片网格
- **场景卡片**：
  1. 实时定位追踪 — GPS/LoRaWAN，全球覆盖
  2. 电子围栏告警 — 手绘多边形/模板，越界自动推送
  3. AI 健康监测 — 发热/消化/发情/疫病 4 场景
  4. 无监督异常检测 — STL+CUSUM 三维联合，误报率低
  5. 设备全生命周期 — 追踪器/胶囊/耳标，在线率监控
  6. 离线地图作业 — 无网络环境正常使用，自动同步
- **动画**：卡片 hover 上浮 4px + 阴影加深
- **交互**：点击卡片展开详情（预留，首版可无）

### ⑤ 架构 — 技术一览

- **背景色**：`#F8F6F0`
- **内容**：左侧 DDD 洋葱架构简图（9 上下文）+ 右侧关键数字面板 + 底部可展开技术栈标签云

- **标签云**：Spring Boot 3.3 / PostgreSQL 16 / Redis 7 / RocketMQ / Flutter / Riverpod / AI Platform

- **目标**：技术读者可深挖，业务读者不必看

- **对比维度**：GPS定位 / 虚拟围栏 / 电池寿命 / 通信方式 / 强制月费 / 健康监测 / AI异常检测 / 无人机巡检 / 设备防盗

## 3. 技术实现

### 技术选型

| 维度 | 选择 | 理由 |
|------|------|------|
| 文件形态 | 单文件 HTML（内联 CSS + JS） | 零依赖，浏览器直接打开，方便分发 |
| CSS 方案 | 原生 CSS（CSS Variables + Grid + Flexbox） | 无需构建工具，响应式用 Media Query |
| JS 方案 | 原生 JS（IntersectionObserver + 少量 DOM 操作） | 仅滚动动画和折叠展开，不需要框架 |
| 图标 | 内联 SVG（简单几何图形）或 Unicode 符号 | 不依赖图标库 |
| 图表 | 无图表库 — 架构图用 CSS/SVG 手绘 | 减少外部依赖 |

### 文件结构

```
docs/marketing/
├── solution-brochure.html          # 解决方案说明书（本次输出）
├── images/                          # 如有截图/Logo 可后加
└── README.md                        # 说明如何部署到静态服务器
```

### 兼容性

- 目标浏览器：Chrome/Firefox/Safari/Edge 最近 2 个大版本
- 不兼容 IE11（Grid 不支持）
- 移动端：响应式，最小宽度 320px

## 4. 内容撰写原则

1. **业务语言为主**：标题和摘要说人话，避免技术黑话
2. **数据说话**：每个论点尽量配具体数字（如 3 年电池、500 头牧场、<1s 告警延迟）
3. **对比驱动**：在优势和定价区块用竞品对比强化价值感
4. **克制克制**：技术区块折叠，默认不展开；不罗列全部功能，挑 6 个核心场景
5. **CTA 贯穿**：Hero → 场景后 → 定价后 → 底部，共 4 处行动入口
