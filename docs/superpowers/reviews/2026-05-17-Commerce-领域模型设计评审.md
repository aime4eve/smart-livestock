## 二、需修改

### 2.1 SubscriptionService.checkHeartbeat — lastHeartbeatAt NPE 防护

**现状**: 首次 provision 后如果未调用 recordHeartbeat()，lastHeartbeatAt 为 null，`lastHeartbeatAt.plus(...)` 会 NPE
**建议**: 二选一

```java
// 方案A：provision 时初始化
ss.lastHeartbeatAt = Instant.now();

// 方案B：checkHeartbeat 中 null 检查
public void checkHeartbeat() {
    if (status != SubscriptionServiceStatus.ACTIVE) return;
    if (lastHeartbeatAt == null) {
        this.status = SubscriptionServiceStatus.GRACE_PERIOD;
        this.registerEvent(new ServiceHeartbeatLostEvent(tenantId));
        return;
    }
    // ...
}
```

推荐方案A（provision 时初始化），语义更清晰。

### 2.2 SubscriptionTier.ENTERPRISE.calculateMonthlyFee — 不应静默返回 0

**现状**: ENTERPRISE 返回 0，调用方可能忽略，生成 0 元账单
**建议**: 抛异常，强制调用方显式处理

```java
public int calculateMonthlyFee(int livestockCount) {
    if (this == ENTERPRISE)
        throw new ApiException(ErrorCode.ENTERPRISE_CUSTOM_PRICING, "Enterprise 需定制计费");
    // ...
}
```

## 三、建议修改

### 3.1 expireTrial() 后状态语义 — ACTIVE vs FREE

**现状**: 试用到期降级为 basic，status 变成 ACTIVE
**问题**: ACTIVE 语义上是"付费订阅生效"，但 basic 是免费层，用户并未付费


- 新增 FREE 状态（TRIAL → FREE），前端可区分付费活跃和免费活跃

### 3.2 Contract — revenueShareRatio 缺少范围校验

**现状**: calculateRevenueShare 直接做乘法，无范围约束
**建议**: 工厂方法或签合同时校验

```java
if (revenueShareRatio.compareTo(BigDecimal.ZERO) < 0
    || revenueShareRatio.compareTo(BigDecimal.ONE) > 0) {
    throw new ApiException(ErrorCode.INVALID_PARAM, "分润比例须在 0~1 之间");
}
```

### 3.3 reactivate() 缺少续费逻辑

**现状**: SUSPENDED → ACTIVE 的 reactivate() 只改状态，不更新 expiresAt
**问题**: 欠费暂停后恢复，计费周期未更新
**建议**: reactivate() 中更新 expiresAt，或新增 renew() 方法

```java
public void reactivate() {
    requireStatus(SubscriptionStatus.SUSPENDED);
    this.status = SubscriptionStatus.ACTIVE;
    // 顺延计费周期
    this.expiresAt = Instant.now().plus(
        "yearly".equals(billingCycle) ? 365 : 30, ChronoUnit.DAYS);
}
```

### 3.4 SubscriptionTier.fenceLimit 与 feature_gates 双 source of truth

**现状**: 枚举中硬编码 fenceLimit（BASIC=3, STANDARD=5, PREMIUM=10），feature_gates 表中也有 fence_management 的 limit_value
**问题**: 两处可能不一致
**建议**: 二选一

- 枚举中去掉 fenceLimit，统一由 feature_gates 表管控

### 3.5 RevenuePeriod.calculate — 前置条件需标注

**现状**: 静态工厂方法不持有 Contract 引用，无法校验合同状态
**建议**: 在方法注释中标注前置条件，ApplicationService 层调用前显式校验

```java
/**
 * 计算分润周期
 * @pre contract.isActive() — 调用方须先校验合同状态
 */
```
