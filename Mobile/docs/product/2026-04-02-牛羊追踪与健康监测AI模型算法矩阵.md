## 一、场景与算法映射总览

### 1.1 核心应用场景

| 场景 | 目标 | 输入数据 | 输出结果 | 价值 |
|------|------|----------|----------|------|
| **定位追踪** | 实时定位、轨迹分析 | GPS坐标、时间戳 | 位置、轨迹、移动距离 | 防盗、放牧管理 |
| **电子围栏** | 越界检测、告警 | GPS坐标、围栏边界 | 越界状态、告警 | 牲畜安全、放牧控制 |
| **体温监测** | 疾病预警 | 瘤胃温度、环境温度 | 体温异常、健康风险 | 早发现、早治疗 |
| **行为分析** | 发情检测、健康评估 | 活动量、采食、反刍 | 行为模式、异常识别 | 繁殖管理、健康管理 |
| **消化监测** | 营养评估、疾病预警 | 蠕动频率、瘤胃pH | 消化状态、健康风险 | 精准饲喂、疾病预防 |
| **繁殖管理** | 发情识别、产犊预测 | 多维数据融合 | 发情期、预产期 | 提高繁殖效率 |

## 三、模型分层架构

### 3.1 三层模型架构

```
┌─────────────────────────────────────────────────────┐
│                    L3 认知决策层                      │
│        LLM Agent | 多轮对话 | 工具调用 | 决策建议      │
│        场景：配种建议、综合诊断、方案推荐              │
├─────────────────────────────────────────────────────┤
│                    L2 深度学习层                      │
│        神经网络 | GPU推理 | 模型>10MB                │
│        场景：发情检测、疾病预测、行为识别             │
├─────────────────────────────────────────────────────┤
│                    L1 轻量模型层                      │
│        传统ML | CPU推理 | 模型<10MB                  │
│        场景：实时检测、边缘计算、快速响应             │
└─────────────────────────────────────────────────────┘
```

### 3.2 模型选择原则

| 场景特点 | 推荐层级 | 理由 |
|----------|----------|------|
| 实时性要求高（<1s） | L1 | 边缘计算，低延迟 |
| 精度要求高 | L2 | 深度学习，高准确率 |
| 需要解释性 | L1/L2 | 可解释模型 |
| 复杂决策 | L3 | LLM 推理能力 |
| 资源受限 | L1 | 轻量模型 |
| 多模态融合 | L2 | 深度学习优势 |

## 四、数据流与模型调用

### 4.1 数据流架构

```
设备端                    边缘网关                   云端
┌─────────┐            ┌─────────┐            ┌─────────┐
│ GPS     │──LoRa─────▶│ 滤波    │──MQTT────▶│ 轨迹分析 │
│ 温度    │            │ 异常检测│            │ 疾病预测 │
│ 加速度  │            │ 行为识别│            │ LLM Agent│
│ 蠕动    │            │ 特征提取│            │         │
└─────────┘            └─────────┘            └─────────┘
    L0                     L1                     L2/L3
```

### 4.2 模型调用流程

**场景1：疾病预警**
```
1. 边缘端：温度异常检测（L1）→ 触发告警
2. 云端：疾病分类模型（L2）→ 疾病类型
3. 云端：LLM Agent（L3）→ 诊断建议
```

**场景2：发情检测**
```
1. 边缘端：行为识别（L1/L2）→ 活动量异常
2. 云端：发情检测模型（L2）→ 发情概率
3. 云端：LLM Agent（L3）→ 配种建议
```

**场景3：电子围栏告警**
```
1. 边缘端：越界检测（L1）→ 实时告警
2. 云端：轨迹分析（L1）→ 移动方向
3. 云端：通知推送（系统）→ 用户接收
```
