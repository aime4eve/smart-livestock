## 二、项目背景与必要性

### 2.1 行业背景

全球畜牧业正在经历从经验养殖到数据驱动精准养殖的转型。欧洲市场尤为突出：

- **欧盟 CAP 政策补贴**：对精准农业技术的补贴支持力度持续加大
- **动物福利法规**：北欧国家动物健康监测要求日益严格，推动市场需求增长
- **GDPR 合规**：数据隐私法规要求本地化存储，为私有部署方案创造差异化空间
- **欧美牧场 IoT 普及率已超 60%**，市场教育成熟

### 2.2 传统牧场管理痛点

传统牧场完全依赖人工经验，缺乏任何技术手段辅助，导致以下核心问题：


| 痛点      | 现状                               | 经济影响                      |
| ------- | -------------------------------- | ------------------------- |
| 围栏成本高   | 铁丝网/木桩建设成本 €500-2000/km，维护人力成本高  | 基础设施投入大                   |
| 疾病发现晚   | 依赖人工巡栏，疾病发现时往往已扩散                | 疫病爆发可能全群扑杀，损失数百万元         |
| 繁殖效率低   | 发情期识别靠经验，漏检率高                    | 每头奶牛延迟配种 21 天损失 ¥630-1050 |
| 盗窃风险    | 偏远牧场缺乏有效监控手段                     | 单头牛价值数千至万元                |
| 人力依赖    | 放牧、巡栏、健康检查全靠人工                   | 人工成本逐年上升                  |
| 人工巡栏效率低 | 大型牧场面积数千亩，人工巡栏耗时长、覆盖面有限，偏远角落难以顾及 | 巡栏周期长，异常发现延迟              |
| 夜间监控空白  | 夜间（22:00-06:00）无人值守，盗窃和越界无法及时发现  | 夜间盗窃损失大，越界牲畜次日才发现         |


> 以上痛点表明，牧场管理迫切需要引入 IoT 技术实现数字化升级。为解决传统牧场的上述痛点，业界通常从以下路径出发：**部署 GPS 追踪设备获取牲畜位置 → 建立虚拟围栏替代物理围栏 → 通过传感器采集健康数据实现预警 → 将数据汇总至云端平台供牧场主决策**。这条路径在逻辑上是完整的，也确实催生了 Halter（新西兰）、Vence（美国）、Nofence（挪威）、Digitanimal（西班牙）等一批先行产品，推动了行业从"靠经验"向"靠数据"的转型。

然而，**这条路径的每一步都面临技术选型的根本性挑战**：通信层用蜂窝还是 LoRaWAN？传感器用外部穿戴设备还是体内植入？设备安全性如何保障？当前竞品在这些关键选择上的取舍，导致方案在通信覆盖、数据质量和防盗能力上均存在结构性短板。以下按"通信层 → 传感器层 → 安全层"逐层剖析，同时说明我方方案如何系统性突破这些瓶颈。

### 2.3 通信层：蜂窝/Wi-Fi 方案的局限性

一个 IoT 系统能否在偏远牧场可靠运行，首先取决于通信技术选型。Halter、Vence、Nofence 等主流竞品均采用蜂窝网络（4G/LTE），部分方案使用短距无线（Wi-Fi/BLE）。通信技术直接决定了系统的覆盖能力、续航时间和成本结构，而蜂窝方案在这一场景下存在以下固有局限：


| 局限            | 具体表现                                           | 影响                         |
| ------------- | ---------------------------------------------- | -------------------------- |
| **偏远山区信号盲区**  | 蜂窝网络在山区、峡谷、林区覆盖薄弱甚至无信号；Wi-Fi 覆盖半径仅数十米，无法满足放牧场景 | 大量牧场面积处于监测空白，牲畜越界/丢失无法及时发现 |
| **频繁更换电池**    | 蜂窝方案设备续航仅 6 个月-1 年，千头牧场每年需大规模回收、更换、重新佩戴        | 人力成本高（每次更换需围捕牲畜）；更换空窗期监测中断 |
| **强制月费锁定**    | 竞品将蜂窝通信费嵌入订阅，采用"设备 + 强制月费"模式                   | 总拥有成本高，客户被锁定无法灵活退出         |
| **恶劣环境可靠性差**  | 蜂窝模块功耗高、发热大，极寒/高温环境下故障率上升                      | 设备在线率不稳定，关键告警可能丢失          |
| **数据延迟与带宽成本** | 蜂窝方案持续上传高频 GPS 数据，流量费用高；偏远区域带宽有限导致数据积压         | 实时性下降，告警延迟；月度通信费用居高不下      |


> **我方方案**：采用 LoRaWAN 通信——5-10km 覆盖半径、3 年超长续航、无通信月费，从根本上突破通信层天花板。固定网关覆盖盲区由**无人机搭载移动 LoRaWAN 网关**动态补盲，实现"固定 + 移动"双层覆盖架构。
>
> 通信层解决后，下一个关键问题是：**设备采集的数据质量是否足够支撑健康预警和 AI 分析？**

### 2.4 传感器层：仅外部穿戴设备的健康监测盲区

即使通信层问题得到解决，当前市场上的 IoT 方案（包括 LoRaWAN 方案如 Digitanimal）普遍仅提供 GPS 耳标或项圈追踪器，缺乏体内植入式传感器，在健康监测层面存在严重盲区：


| 盲区             | 具体表现                                                       | 影响                                                     |
| -------------- | ---------------------------------------------------------- | ------------------------------------------------------ |
| **无法监测核心体温**   | 耳标/项圈只能采集体表温度，受环境温度、日照、风速干扰大，测量精度远低于瘤胃内部温度（±0.1°C）         | 体温异常检测误报率高，无法建立可靠的个体基线，疾病预警可信度低                        |
| **消化系统疾病盲区**   | 无蠕动传感器，无法监测瘤胃蠕动频率和强度                                       | 瘤胃迟缓、积食、胀气等消化系统疾病（占牛群疾病 30-40%）无法早期发现，干预延迟导致治疗成本增加 60% |
| **发情识别准确率低**   | 仅靠 GPS 移动距离 + 加速度计步数判断发情，缺乏体温变化维度（发情期体温升高 0.2-0.5°C 是关键指标） | 多指标融合缺少核心维度，发情漏检率高，直接影响繁殖效率和产奶收益                       |
| **体表设备易受干扰**   | 耳标/项圈暴露在外，冬季低温导致电池衰减加速、传感器精度下降；夏季高温导致体表温度失真                | 数据质量季节性波动，冬季续航缩短、夏季预警失准                                |
| **缺乏体内持续监测**   | 外部设备无法提供持续、稳定的生理参数，只能做间歇性快照                                | 无法构建完整的 24h 生理节律曲线，AI 模型缺乏高质量训练数据                      |
| **缺乏 AI 分析能力** | 现有方案停留在"数据采集 + 展示"层面，无行为分析、趋势预测、智能决策                       | 牧场主仍需人工解读数据，未能真正释放数据价值                                 |


> **我方方案**：瘤胃胶囊作为健康监测的"金标准"传感器，提供体温（±0.1°C）+ 蠕动频率的体内持续监测。胶囊与追踪器构成双设备架构，实现体内体外数据融合互补，为数智孪生 AI 场景提供高质量数据底座。
>
> 传感器层解决后，还剩一个关键问题：**外部佩戴的追踪器本身可被拆除，如何确保系统不被盗畜者绕过？**

### 2.5 安全层：可拆除设备的防盗盲区

上述所有方案（蜂窝/LoRaWAN、耳标/项圈）的追踪设备均为外部佩戴，存在被人为拆卸或意外脱落的风险。这是当前行业的共性安全盲区：


| 盲区           | 具体表现                                                  | 影响                              |
| ------------ | ----------------------------------------------------- | ------------------------------- |
| **设备可被轻易拆除** | 项圈卡扣可被工具剪断，耳标可被徒手拔出；盗畜者拆除设备后牲畜即"失联"                   | 牲畜被盗后无法追踪定位，相当于防盗系统被直接绕过        |
| **设备脱落无感知**  | 现有方案无关联监测机制，耳标/项圈脱落后系统仅显示"离线"，牧场主无法区分设备故障、信号丢失还是设备丢失  | 告警信息模糊，牧场主无法判断是否需要紧急处理，错失黄金追回时间 |
| **离线即失联**    | 外部设备一旦脱落，牲畜与平台彻底断开连接，无备用通信通道                          | 被盗牲畜无法被追踪、被找回，设备防盗形同虚设          |
| **夜间盗窃难发现**  | 非放牧时段（22:00-06:00）无人值守，盗畜者拆除设备后驱离牲畜，牧场主次日才发现          | 单次盗窃可能损失数十头，经济损失数万至数十万元         |
| **追回成本高**    | 发现丢失后需人工搜寻，缺乏最后已知位置、移动方向等线索；偏远山区固定 LoRaWAN 网关无法覆盖全部区域 | 找回率低，偏远山区大型牧场找回概率不足 30%         |


> **我方方案**：瘤胃胶囊驻留在瘤胃内，无法被外部拆除。胶囊与追踪器通过蓝牙关联——当蓝牙链接断开超过 5 分钟，胶囊主动经 LoRaWAN 上报"追踪器丢失告警"并持续提供定位。即使追踪器被拆除，胶囊仍是"不可移除的信标"，从根本上杜绝防盗系统被绕过的可能性。进一步地，**无人机搭载移动 LoRaWAN 网关可飞至疑似区域主动搜索胶囊/追踪器信号**，将"被动等待告警"升级为"主动空中搜寻"，大幅提升丢失牲畜找回率。
>
> 正是胶囊同时解决了传感器层（健康监测）和安全层（防盗兜底）两个问题，而无人机移动网关进一步延伸了安全层的搜索半径，体现了双设备架构 + 空地协同的核心价值。

### 2.6 立项必要性

综合以上四层分析，本项目立项的必要性在于：

1. **市场需求明确**：传统牧场痛点迫切，欧洲精准农业市场规模持续增长，虚拟围栏 + 健康监测成为刚需
2. **通信层突破**：LoRaWAN 以 5-10km 覆盖 + 3 年续航 + 无通信月费，解决蜂窝方案的结构性劣势
3. **传感器层创新**：瘤胃胶囊提供体内金标准数据，填补行业健康监测空白，为数智孪生 AI 奠定数据基础
4. **安全层差异化**：胶囊与追踪器蓝牙关联防盗，解决外部设备可被拆除的行业共性难题
5. **商业模式验证**：竞品 Halter、Nofence 已验证"设备 + 订阅"模式可行性，我方以"可选订阅"进一步降低客户门槛
6. **无人机巡检扩展**：集成第三方工业无人机 SDK，实现自动巡航、告警联动追踪、声音驱赶与孪生平台融合。无人机同时搭载移动 LoRaWAN 网关，一机双能：既补足 LoRaWAN 固定网关的覆盖盲区，又能主动飞至丢失动物附近搜索胶囊/追踪器信号，构建"地面 IoT + 空中无人机"立体监控与通信体系

## 四、产品定义与技术方案

### 4.1 产品定义

面向牧场主的牲畜智能管理平台，通过 IoT 设备和 AI 模型实现：

1. **定位管控** — GPS 实时追踪 + 虚拟电子围栏 + 越界告警
2. **健康预警** — 瘤胃温度基线偏离检测 + 消化蠕动异常监测
3. **行为分析** — 发情识别 + 疫病防控 + 群体趋势分析

### 4.2 系统架构

```
终端层
├── Mobile App（iOS/Android）
├── Operation PC 管理端
└── API Gateway（REST/MQTT）

服务层（后端云服务：智能体支撑平台）
├── 业务中台（用户注册 + 订阅支付）
├── 数据中台（告警引擎 + 分析引擎）
├── 物联网中台（设备管理 + 数据接入）
└── AI中台（L1 轻量模型 + L2 深度学习 + L3 认知决策）

设备层
├── 动物追踪器（GPS + 加速度计 + LoRaWAN + BLE）
├── 瘤胃胶囊（温度 + 蠕动 + BLE + LoRaWAN 备用）
├── 巡检无人机（第三方工业无人机 + SDK，视频/红外/扩音器 + 移动 LoRaWAN 网关）
└── LoRaWAN 网关（固定基站）
```

### 4.3 核心技术选型


| 层级    | 技术方案                                   | 选型理由                |
| ----- | -------------------------------------- | ------------------- |
| 移动端   | Flutter (Dart)                         | 跨平台单一代码库，降低开发成本     |
| PC 端  | VUE3                                   | 企业级管理后台，生态成熟        |
| 云服务端  | 智能体支撑平台                                | 高性能异步框架，ML 生态集成     |
| -     | 数据库：PostgreSQL + TDengine              | 关系数据 + 时序数据双引擎      |
| -     | 缓存：Redis                               | 会话管理、热点查询、限流        |
| -     | 实时通信：MQTT (EMQX)                       | 轻量级、IoT 标准协议        |
| -     | AI 模型：TensorFlow/PyTorch/XGBoost       | L1-L3 三层模型架构        |
| -     | 地图服务：Mapbox                            | 离线支持、围栏绘制           |
| 无人机   | 第三方工业无人机 SDK（大疆 DJI Mobile SDK / MSDK） | 成熟飞控生态，降低研发风险       |
| 无人机载荷 | 轻量化 LoRaWAN 网关模块（Semtech SX1302）       | 无人机搭载移动网关，补盲 + 主动搜寻 |


### 4.4 AI 模型三层架构


| 层级          | 技术                           | 应用场景           | 推理环境      |
| ----------- | ---------------------------- | -------------- | --------- |
| **L1 轻量模型** | scikit-learn, TFLite         | 实时异常检测、边缘计算    | CPU / 云端  |
| **L2 深度学习** | TensorFlow, PyTorch, XGBoost | 发情检测、疾病预测、行为识别 | GPU / 云端  |
| **L3 认知决策** | LLM Agent                    | 综合诊断、配种建议、方案推荐 | 硅基流动 / 云端 |

## 五、研发内容与功能范围

### 5.1 功能模块总览

### 5.2 MVP（V1.0）核心交付

MVP 阶段聚焦定位管控 + 基础健康 + 租户管理三大能力：

- **GPS 定位**：2-5m 精度，5-15 分钟上报频率
- **虚拟围栏**：多边形/圆形绘制，进入/离开/限制三种类型，时间策略
- **越界告警**：实时推送，端到端时延 < 60s（P95）
- **发热预警**：个体基线学习（7 天均值），±0.5°C 异常检测
- **消化监测**：蠕动频率基线对比，异常分级告警
- **发情识别**：GPS + 加速度计 + 温度多指标融合评分
- **疫病防控**：群体指标偏移预警，GPS 接触链路追踪
- **告警状态机**：pending → acknowledged → handled → archived
- **租户管理**：开通/禁用、设备 License 配额管理
