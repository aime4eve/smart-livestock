# 智慧畜牧 App 高保真评审脚本

> **用途**：用于演示高保真 UI/UX 版本，确认视觉、交互、角色边界与 mock/live 切换说明。  
> **运行模式**：默认 `APP_MODE=mock`；如需查看联调壳，使用 `APP_MODE=live` 启动，但当前 live 仍以内置 mock datasource 兜底。

---

## 1. 演示前准备

- 进入工程：`cd mobile_app`
- 安装依赖：`flutter pub get`
- 默认 Mock 演示：
  - `flutter run`
- Live 联调壳演示：
  - `flutter run --dart-define=APP_MODE=live`

建议先用 `mock` 跑完整流程，再切 `live` 说明架构切换点。

---

## 2. 本轮评审重点

- **高保真**：设计系统、卡片层次、状态芯片、页面一致性
- **虚拟围栏**：地图主画布、围栏工具栏、图层控制、模板表达
- **越界 / 低电 / 失联**：P0 告警三类视觉层次与操作流
- **角色边界**：`worker` / `owner` / `ops` 的菜单与操作差异
- **场景**：正常、围栏越界、设备低电、信号丢失、离线缓存

---

## 3. 10 分钟演示顺序

### 3.1 登录与模式说明（1 分钟）

- 打开高保真登录页，说明当前为 `Mock 场景`
- 展示角色切换：
  - `worker`
  - `owner`
  - `ops`
- 说明：页面层不分支 mock/live，统一走 `Page -> Controller -> Repository -> DataSource`

### 3.2 Dashboard（1.5 分钟）

- 展示牧场头部、四张指标卡、快捷入口
- 说明高保真视觉系统已统一到首页

### 3.3 地图与虚拟围栏（3 分钟）

- 进入地图页
- 展示工具栏：选择 / 绘制围栏 / 编辑 / 删除 / 测量
- 展示图层：围栏 / 牲畜 / 告警 / 轨迹
- 切换牲畜与时间区间，说明虚拟围栏场景和离线缓存策略

### 3.4 告警中心（2 分钟）

- 展示三类 P0 告警：
  - 越界
  - 低电
  - 失联
- 依次演示：
  - 确认
  - 处理
  - 归档

### 3.5 后台 / 我的 / 权限边界（2.5 分钟）

- `owner`：可见后台入口、围栏编辑、license 调整
- `worker`：不可见后台，围栏页无编辑动作
- `ops`：直达运维后台，不显示业务底栏
