# 智慧畜牧 App — 低保真 Demo 客户评审脚本

> **用途**：现场或远程投屏演示 `mobile_app` 可点击 Demo，约 **10 分钟**完成「入口 + 流程 + 角色 + 异常态」确认。  
> **说明**：本 Demo **不接真实后端**，数据与状态为本地模拟；若与客户心理预期冲突，以会后「保留 / 修改 / 新增 / 冻结」四类结论为准。

---

## 演示前（2 分钟）

- 环境：`Flutter` 已安装；在工程根目录执行：
  - `cd mobile_app && flutter pub get`
  - `cd mobile_app && flutter run`（任选 iOS / Android / macOS / Chrome 目标）
- 说明三句话：
  1. 这是**需求确认用**交互壳，不是视觉终稿。  
  2. 登录页 **worker / owner / ops** 为**演示角色开关**（正式版由账号与接口决定角色）。  
  3. 各页顶部 **状态切换** 用于看加载/空/错/无权限/离线，**不接真实网络**。

---

## 六类界面状态（约 2 分钟，可穿插在各流程中演示）

在 **看板 / 地图 / 告警 / 我的 / 围栏 / 后台（owner）** 任一页，将状态下拉改为：

| 状态 | 客户需确认 |
|------|------------|
| normal | 默认可用布局与文案是否够用 |
| loading | 是否有明确「加载中」感知 |
| empty | 空态提示与引导是否可接受 |
| error | 错误提示与重试（或地图列表回退）是否合理 |
| forbidden | 无权限文案是否清晰 |
| offline | 离线提示与「最近同步」类信息是否需加强 |

**地图页专项**：在 **error** 态下展示**列表回退**（模拟地图不可用），请客户确认是否接受该产品策略。

---

## 流程 1：登录与角色分流（约 1 分钟）

1. **owner**：登录后底部应有 **看板 / 地图 / 告警 / 我的 / 围栏 / 后台**。  
2. **worker**：登录后**无「后台」**，围栏页**无编辑类按钮**。  
3. **ops**：登录后**仅租户后台**，无牧场业务底栏。

**确认问题**：三类角色与贵司实际岗位是否一致？是否需要增删菜单？

---

## 流程 2：地图筛选与回放区间（约 2 分钟）

1. 使用 **owner** 进入 **地图**。  
2. **牲畜筛选** 切换耳标；**24h / 7d / 30d** 切换区间，看图占位区摘要是否随选择变化。  
3. （可选）切 **error** 态看列表回退。

**确认问题**：筛选维度、默认区间、失败回退是否满足日常运维？

---

## 流程 3：告警闭环（约 2 分钟）

1. **owner** 进入 **告警**：依次 **确认 → 处理 → 归档**，观察状态文案变化。  
2. （可选）**worker** 进入 **告警**：仅应出现 **确认**，无处理/归档。

**确认问题**：状态是否缺「误报」「转派」等？批量处理入口是否保留？

---

## 流程 4：围栏与租户（约 2 分钟）

1. **owner** → **围栏**：演示 **新增 / 编辑 / 删除** 入口；点击编辑后本地提示「已保存」类反馈。  
2. **owner** → **后台**：演示 **开通 / 禁用·启用 / 调整 license**；点击调整 license 后本地提示。  
3. **ops** 整体验：仅后台，与 business 互斥。

**确认问题**：谁有权改围栏与租户 license？是否需要审批流？
