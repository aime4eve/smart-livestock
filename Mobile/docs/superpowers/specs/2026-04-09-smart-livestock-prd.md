## 一、产品概述

### 1.1 产品定位

面向牧场主的牲畜智能管理平台，通过 IoT 设备、AI 模型和无人机巡检实现牲畜**定位管控**、**健康预警**、**行为分析**和**空地协同巡检**，帮助牧场主从经验养殖转向数据驱动的精准养殖。

### 1.2 产品愿景

成为全球领先的畜牧业**数智孪生**平台。从数据孪生（实时监测）到模型孪生（AI 预警）再到智能孪生（自主决策闭环），覆盖牲畜全生命周期管理。

### 1.3 目标用户

| 客户类型 | 规模 | 付费能力 | 核心需求 |
|----------|------|----------|----------|
| **小型牧场** | 50-200 头 | 中 | 定位防盗、基础健康 |
| **中型牧场** | 200-1000 头 | 较高 | 繁殖管理、疾病预警 |
| **大型牧场** | 1000-5000 头 | 高 | 全面管理、降本增效 |
| **超大型牧场** | 5000+ 头 | 很高 | 集团化管理、数据分析 |

### 1.4 用户角色

| 角色 | 典型用户 | 核心诉求 |
|------|---------|---------|
| **牧场主/养殖户** | 中大型牧场经营者 | 群体概览、异常预警、减少人工巡栏 |
| **牧工/放牧员** | 日常放牧人员 | 实时位置查看、围栏告警处理 |
| **兽医/技术员** | 兽医站、畜牧技术员 | 个体健康分析、历史数据追溯 |
| **平台运维**（仅云端） | 区域云运维团队 | 租户开通、运行监控、故障处理 |

### 1.5 部署模式

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| **私有化部署** | 大型养殖场 | 数据完全私有，App 内网/VPN 访问 |
| **区域云端部署** | 中小型养殖户 | 多租户共享，App 公网访问 |

## 二、市场分析

### 2.1 行业背景

全球畜牧业正在经历从经验养殖到数据驱动精准养殖的转型。欧洲市场尤为突出：

- **欧盟 CAP 政策补贴**：对精准农业技术的补贴支持力度持续加大
- **动物福利法规**：北欧国家动物健康监测要求日益严格，推动市场需求增长
- **GDPR 合规**：数据隐私法规要求本地化存储，为私有部署方案创造差异化空间
- **欧美牧场 IoT 普及率已超 60%**，市场教育成熟

传统牧场完全依赖人工经验，缺乏任何技术手段辅助，导致以下核心痛点：

#### 2.1.1 传统围栏痛点

| 痛点 | 问题 | 影响 |
|------|------|------|
| 建设成本 | 铁丝网、木桩、水泥桩 | 每公里 €500-2000 |
| 维护成本 | 定期检查、修复破损围栏 | 人工成本高 |
| 灵活性差 | 固定围栏无法快速调整牧场范围 | 管理不灵活 |
| 环境限制 | 山地、湿地等地形难以建设 | 牧场利用受限 |
| 牲畜伤害 | 牲畜触碰围栏可能受伤 | 损失风险 |
| 人工巡栏效率低 | 大型牧场面积数千亩，人工巡栏耗时长、覆盖面有限，偏远角落难以顾及 | 巡栏周期长，异常发现延迟 |
| 夜间监控空白 | 夜间（22:00-06:00）无人值守，盗窃和越界无法及时发现 | 夜间盗窃损失大，越界牲畜次日才发现 |

#### 2.1.2 通信层局限：蜂窝/Wi-Fi 方案的结构性劣势

一个 IoT 系统能否在偏远牧场可靠运行，首先取决于通信技术选型。Halter、Vence、Nofence 等主流竞品均采用蜂窝网络（4G/LTE），部分方案使用短距无线（Wi-Fi/BLE）。

| 局限 | 具体表现 | 影响 |
|------|----------|------|
| 偏远山区信号盲区 | 蜂窝网络在山区、峡谷、林区覆盖薄弱甚至无信号；Wi-Fi 覆盖半径仅数十米 | 大量牧场面积处于监测空白 |
| 频繁更换电池 | 蜂窝方案设备续航仅 6 个月-1 年，千头牧场每年需大规模回收更换 | 人力成本高，更换空窗期监测中断 |
| 强制月费锁定 | 竞品将蜂窝通信费嵌入订阅，采用"设备 + 强制月费"模式 | 总拥有成本高，客户被锁定 |
| 恶劣环境可靠性差 | 蜂窝模块功耗高、发热大，极寒/高温环境下故障率上升 | 设备在线率不稳定，关键告警可能丢失 |

> **我方对策**：采用 LoRaWAN 通信——5-10km 覆盖半径、3 年超长续航、无通信月费。固定网关覆盖盲区由无人机搭载移动 LoRaWAN 网关动态补盲，实现"固定 + 移动"双层覆盖架构。

#### 2.1.3 传感器层盲区：仅外部穿戴设备的健康监测局限

即使通信层问题得到解决，当前市场上的 IoT 方案（包括 LoRaWAN 方案如 Digitanimal）普遍仅提供 GPS 耳标或项圈追踪器，缺乏体内植入式传感器，在健康监测层面存在严重盲区。

| 盲区 | 具体表现 | 影响 |
|------|----------|------|
| 无法监测核心体温 | 耳标/项圈只能采集体表温度，受环境温度、日照、风速干扰大 | 体温异常检测误报率高，无法建立可靠个体基线 |
| 消化系统疾病盲区 | 无蠕动传感器，无法监测瘤胃蠕动频率和强度 | 消化系统疾病（占牛群疾病 30-40%）无法早期发现 |
| 发情识别准确率低 | 仅靠 GPS 移动距离 + 加速度计判断，缺乏体温变化维度 | 多指标融合缺少核心维度，发情漏检率高 |
| 体表设备易受干扰 | 冬季低温导致电池衰减加速、夏季高温导致体表温度失真 | 数据质量季节性波动 |
| 缺乏 AI 分析能力 | 停留在"数据采集 + 展示"层面，无行为分析、趋势预测 | 牧场主仍需人工解读数据 |

> **我方对策**：瘤胃胶囊提供体温（±0.1°C）+ 蠕动频率的体内持续监测，与追踪器构成双设备架构，实现体内体外数据融合互补，为数智孪生 AI 场景提供高质量数据底座。

#### 2.1.4 安全层盲区：可拆除设备的防盗局限

上述所有方案的追踪设备均为外部佩戴，存在被人为拆卸或意外脱落的风险。

| 盲区 | 具体表现 | 影响 |
|------|----------|------|
| 设备可被轻易拆除 | 项圈卡扣可被工具剪断，耳标可被徒手拔出 | 牲畜被盗后无法追踪定位，防盗系统被直接绕过 |
| 设备脱落无感知 | 无关联监测机制，脱落后系统仅显示"离线" | 牧场主无法区分设备故障还是设备丢失 |
| 离线即失联 | 外部设备一旦脱落，牲畜与平台彻底断开连接 | 被盗牲畜无法被追踪找回 |

> **我方对策**：瘤胃胶囊驻留在瘤胃内，无法被外部拆除。胶囊与追踪器通过蓝牙关联——当蓝牙链接断开超过 5 分钟，胶囊主动经 LoRaWAN 上报"追踪器丢失告警"并持续提供定位。即使追踪器被拆除，胶囊仍是"不可移除的信标"。无人机搭载移动 LoRaWAN 网关可飞至疑似区域主动搜索胶囊/追踪器信号，将"被动等待告警"升级为"主动空中搜寻"。

### 4.3 设备通信架构

```
正常模式（蓝牙畅通）：
  瘤胃胶囊 ──BLE──→ 动物追踪器 ──LoRaWAN──→ 固定网关 → 平台

备用模式（蓝牙断开）：
  瘤胃胶囊 ──LoRaWAN──→ 固定网关 → 平台
  动物追踪器 ──LoRaWAN──→ 固定网关 → 平台

防盗告警（蓝牙断开超时）：
  瘤胃胶囊检测蓝牙链接断开 > 5 分钟
  → 胶囊启动 LoRaWAN
  → 上报"动物追踪器丢失告警"
  → 平台推送告警至牧场主
  → 可选：触发无人机起飞搜寻

移动网关补盲（固定网关盲区）：
  IoT 设备 ──LoRaWAN──→ 无人机搭载移动网关 → 平台
  （无人机飞至目标区域主动搜索胶囊/追踪器信号）

告警联动追踪（越界/丢失）：
  告警触发 → 无人机自动起飞 → 飞至告警位置
  → 视频回传 + 移动网关搜寻 + 声音驱赶回栏
```

### 5.3 模块 B：数智孪生

#### 5.3.2 发热预警

**数据来源**：瘤胃胶囊温度传感器，每 30 分钟一次。

| 功能 | 说明 |
|------|------|
| 基线温度自动学习 | 每头牲畜个体基线（7 天均值） |

## 八、AI 模型算法矩阵

### 8.1 场景与算法映射

| 场景 | L1 算法 | L2 模型 | L3 能力 |
|------|---------|---------|---------|
| **定位追踪** | 卡尔曼滤波、DBSCAN 聚类 | LSTM 轨迹预测 | — |
| **电子围栏** | 点在多边形内判断、射线法 | 围栏推荐算法 | — |
| **体温监测** | 统计异常检测、线性回归 | XGBoost 疾病分类 | 综合诊断 |
| **行为分析** | 随机森林、规则引擎 | 1D-CNN 行为识别、发情检测 | 行为解读 |
| **消化监测** | 频域分析、SVM | 营养评估模型 | — |
| **繁殖管理** | 统计方法 | LSTM 产犊预测 | 配种建议 |

### 8.2 模型调用流程

**疾病预警**：
```
边缘端：温度异常检测（L1）→ 触发告警
云端：疾病分类模型（L2）→ 疾病类型
云端：LLM Agent（L3）→ 诊断建议
```

**发情检测**：
```
边缘端：行为识别（L1/L2）→ 活动量异常
云端：发情检测模型（L2）→ 发情概率
云端：LLM Agent（L3）→ 配种建议
```

**电子围栏告警**：
```
边缘端：越界检测（L1）→ 实时告警
云端：轨迹分析（L1）→ 移动方向
系统：通知推送 → 用户接收
```

### 8.3 性能指标

| 场景 | 算法 | 准确率 | 召回率 | 响应时间 |
|------|------|--------|--------|----------|
| 体温异常 | 统计方法 | 85% | 90% | <1s |
| 发情检测 | XGBoost | 85% | 80% | <5s |
| 行为识别 | 1D-CNN | 90% | 88% | <2s |
| 越界检测 | 几何算法 | 99% | 99% | <100ms |

## 十一、技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| LoRaWAN 信号覆盖不足 | 定位不准 | 中 | 现场勘测，合理布站 |
| 标签电池续航 | 数据丢失 | 中 | 选择低功耗标签，定期更换 |
| GPS 定位误差（5-10m） | 误告警 | 高 | 设置围栏缓冲区（10m） |
| 瘤胃胶囊排出 | 健康数据丢失 | 低 | 定期检查，及时补充 |
| ML 模型准确率 | 误报/漏报 | 中 | 持续训练优化，阈值可调 |
| 跨平台适配问题 | UI 不一致 | 低 | Flutter 原生渲染，优先测试 |
| 多语言翻译质量 | 用户体验差 | 低 | 专业翻译 + 本地化测试 |
| 地图 SDK 选型与离线支持 | 地图功能受限 | 中 | 评估 Mapbox（离线能力强） |
| 无人机 SDK 兼容性与更新风险 | 功能受限 | 中 | 抽象飞控接口层，隔离 SDK 变更影响；跟进多厂商方案 |
| 无人机载荷功耗与重量限制 | 续航不足 | 中 | 选用轻量化 SX1302 模块，优化天线设计，限制巡航时长适配载荷功耗 |
| 无人机飞行法规限制 | 部署受限 | 中 | 提前调研目标市场无人机法规（EASA/CAAC），申请必要飞行许可 |
| 硬件供应链不稳定 | 生产延迟 | 中 | 多供应商备选，关键器件备库 |
| 欧洲市场准入合规 | 市场进入延迟 | 中 | 提前启动 CE 认证、GDPR 合规审计 |
| 用户付费意愿低 | 收入不达预期 | 中 | Discovery 免费版引流，ROI 数据佐证 |
