## 一、产品概述

### 1.1 产品定位

面向牧场主的牲畜智能管理平台，通过 IoT 设备和 AI 模型实现牲畜**定位管控**、**健康预警**和**行为分析**，帮助牧场主从经验养殖转向数据驱动的精准养殖。

### 1.2 产品愿景

成为全球领先的畜牧业**数智孪生**平台。从数据孪生（实时监测）到模型孪生（AI 预警）再到智能孪生（自主决策闭环），覆盖牲畜全生命周期管理。

### 1.3 目标用户

| 客户类型 | 规模 | 付费能力 | 核心需求 |
|----------|------|----------|----------|
| **小型牧场** | 50-200 头 | 中 | 定位防盗、基础健康 |
| **中型牧场** | 200-1000 头 | 较高 | 繁殖管理、疾病预警 |
| **大型牧场** | 1000-5000 头 | 高 | 全面管理、降本增效 |
| **超大型牧场** | 5000+ 头 | 很高 | 集团化管理、数据分析 |

### 1.4 用户角色

| 角色 | 典型用户 | 核心诉求 |
|------|---------|---------|
| **牧场主/养殖户** | 中大型牧场经营者 | 群体概览、异常预警、减少人工巡栏 |
| **牧工/放牧员** | 日常放牧人员 | 实时位置查看、围栏告警处理 |
| **兽医/技术员** | 兽医站、畜牧技术员 | 个体健康分析、历史数据追溯 |
| **平台运维**（仅云端） | 区域云运维团队 | 租户开通、运行监控、故障处理 |

### 1.5 部署模式

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| **私有化部署** | 大型养殖场 | 数据完全私有，App 内网/VPN 访问 |
| **区域云端部署** | 中小型养殖户 | 多租户共享，App 公网访问 |

### 1.6 核心价值主张

| 价值点 | 说明 |
|--------|------|
| **LoRaWAN 低功耗** | 3 年电池寿命，远超竞品 6 个月-1 年 |
| **偏远山区覆盖** | 不依赖蜂窝网络，LoRa 覆盖 5-10 公里 |
| **数智孪生** | 4 大孪生场景（发热/消化/发情/疫病）覆盖牲畜全生命周期；另有 GPS 围栏与草场灌溉场景规划中 |
| **防盗能力** | 瘤胃胶囊与追踪器蓝牙关联，链接断开超 5 分钟时胶囊主动上报追踪器丢失告警 |
| **灵活定价** | 一次性购买设备 + 可选订阅增值服务 |

## 二、市场分析

### 2.1 行业背景

欧洲畜牧业智能化趋势显著：欧盟 CAP 对精准农业技术的补贴支持、GDPR 合规要求、北欧国家动物福利法规推动健康监测需求增长。欧美牧场 IoT 普及率超 60%。

### 2.2 传统围栏痛点

| 痛点 | 问题 | 影响 |
|------|------|------|
| 建设成本 | 铁丝网、木桩、水泥桩 | 每公里 €500-2000 |
| 维护成本 | 定期检查、修复破损围栏 | 人工成本高 |
| 灵活性差 | 固定围栏无法快速调整牧场范围 | 管理不灵活 |
| 环境限制 | 山地、湿地等地形难以建设 | 牧场利用受限 |
| 牲畜伤害 | 牲畜触碰围栏可能受伤 | 损失风险 |

### 2.3 竞品分析

| 竞品 | 国家 | 技术路线 | 核心功能 | 定价模式 |
|------|------|----------|----------|----------|
| **Halter** | 新西兰 | 蜂窝网络 | 虚拟围栏 + 健康监测 | 设备 + 月费 |
| **Vence** | 美国 | 蜂窝网络 | 虚拟围栏 + 牧场管理 | 设备 + 月费（SaaS） |
| **Nofence** | 挪威 | 4G/LTE | 虚拟围栏 | 设备 + 月费 |
| **Digitanimal** | 西班牙 | LoRaWAN | 牲畜追踪 + 活动监测 | 设备销售 |

### 4.2 IoT 设备

#### 动物追踪器

| 属性 | 规格 |
|------|------|
| 形态 | 项圈 / 耳标 |
| GPS 模块 | 支持北斗/GPS/GLONASS，2-5 米精度 |
| 加速度计 | 三轴，用于步态分类和行为识别 |
| LoRa 模块 | 868MHz（欧洲），覆盖 5-10 公里 |
| 蓝牙 | BLE，与瘤胃胶囊通信 |
| 电池 | 锂亚硫酰氯（Li-SOCl2），3 年续航 |
| 外壳 | IP67 防护等级 |
| 提醒方式 | 声音 + 振动（虚拟围栏引导） |

#### 瘤胃胶囊

| 属性 | 规格 |
|------|------|
| 形态 | 胶囊（口服投喂） |
| 温度传感器 | 瘤胃温度监测，精度 ±0.1°C |
| 蠕动传感器 | 瘤胃蠕动频率和强度监测 |
| 蓝牙（BLE） | 与动物追踪器通信，优先数据传输路径 |
| LoRaWAN | 备用通信，蓝牙断开时自动启动 |
| 电池 | 2-3 年寿命（一次性） |

### 4.3 设备通信架构

```
正常模式（蓝牙畅通）：
  瘤胃胶囊 ──BLE──→ 动物追踪器 ──LoRaWAN──→ 网关 → 平台

备用模式（蓝牙断开）：
  瘤胃胶囊 ──LoRaWAN──→ 网关 → 平台
  动物追踪器 ──LoRaWAN──→ 网关 → 平台

防盗告警（蓝牙断开超时）：
  瘤胃胶囊检测蓝牙链接断开 > 5 分钟
  → 胶囊启动 LoRaWAN
  → 上报"动物追踪器丢失告警"
  → 平台推送告警至牧场主
```

#### 5.3.2 发热预警

**数据来源**：瘤胃胶囊温度传感器，每 30 分钟一次。

## 八、AI 模型算法矩阵

### 8.1 场景与算法映射

| 场景 | L1 算法 | L2 模型 | L3 能力 |
|------|---------|---------|---------|
| **定位追踪** | 卡尔曼滤波、DBSCAN 聚类 | LSTM 轨迹预测 | — |
| **电子围栏** | 点在多边形内判断、射线法 | 围栏推荐算法 | — |
| **体温监测** | 统计异常检测、线性回归 | XGBoost 疾病分类 | 综合诊断 |
| **行为分析** | 随机森林、规则引擎 | 1D-CNN 行为识别、发情检测 | 行为解读 |
| **消化监测** | 频域分析、SVM | 营养评估模型 | — |
| **繁殖管理** | 统计方法 | LSTM 产犊预测 | 配种建议 |

### 8.2 模型调用流程

**疾病预警**：
```
边缘端：温度异常检测（L1）→ 触发告警
云端：疾病分类模型（L2）→ 疾病类型
云端：LLM Agent（L3）→ 诊断建议
```

**发情检测**：
```
边缘端：行为识别（L1/L2）→ 活动量异常
云端：发情检测模型（L2）→ 发情概率
云端：LLM Agent（L3）→ 配种建议
```

**电子围栏告警**：
```
边缘端：越界检测（L1）→ 实时告警
云端：轨迹分析（L1）→ 移动方向
系统：通知推送 → 用户接收
```

### 8.3 性能指标

| 场景 | 算法 | 准确率 | 召回率 | 响应时间 |
|------|------|--------|--------|----------|
| 体温异常 | 统计方法 | 85% | 90% | <1s |
| 发情检测 | XGBoost | 85% | 80% | <5s |
| 行为识别 | 1D-CNN | 90% | 88% | <2s |
| 越界检测 | 几何算法 | 99% | 99% | <100ms |

## 十一、技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| LoRaWAN 信号覆盖不足 | 定位不准 | 中 | 现场勘测，合理布站 |
| 标签电池续航 | 数据丢失 | 中 | 选择低功耗标签，定期更换 |
| GPS 定位误差（5-10m） | 误告警 | 高 | 设置围栏缓冲区（10m） |
| 瘤胃胶囊排出 | 健康数据丢失 | 低 | 定期检查，及时补充 |
| ML 模型准确率 | 误报/漏报 | 中 | 持续训练优化，阈值可调 |
| 跨平台适配问题 | UI 不一致 | 低 | Flutter 原生渲染，优先测试 |
| 多语言翻译质量 | 用户体验差 | 低 | 专业翻译 + 本地化测试 |
| 地图 SDK 选型与离线支持 | 地图功能受限 | 中 | 评估 Mapbox（离线能力强） |
