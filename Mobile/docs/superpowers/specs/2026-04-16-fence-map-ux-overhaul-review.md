# 围栏地图交互体验重构设计评审意见

> 评审对象：`2026-04-16-fence-map-ux-overhaul-design.md`
> 评审日期：2026-04-16
> 评审结论：需修改后再进入实现

## 高严重度问题

1. 像素容差定义与实现位置不一致，存在缩放失真风险  
   - 文档要求边界命中容差为 40px（屏幕空间），但文件变更计划放在 `domain` 的 `LatLng` 检测函数中。  
   - 若无地图投影上下文，无法稳定表达像素距离，缩放变化会导致命中行为不一致。

2. 文档内部存在变更声明冲突  
   - 一处写明扩展 `fence_polygon_contains.dart`（`domain` 层变更）。  
   - 另一处写明“所有 `domain/` 和 `data/` 层代码不变”。  
   - 该冲突会导致评审口径、任务拆解和实现边界不一致。

## 中严重度问题

1. 手势原则与手势表不一致  
   - 目标中写“单指只操作顶点”。  
   - 手势表中又定义插点/删点工具下单指拖动为平移地图。  
   - 建议统一为单一规则，并同步到目标、交互细则与验收标准。

2. 视觉规范未对齐主题 token 约束  
   - 文档使用 `Colors.black45`。  
   - 项目规范要求使用 `AppColors/AppSpacing/AppTypography`，建议改为主题 token 表达。

3. 测试影响描述不足以覆盖验收项  
   - 当前仅明确一个点击高亮测试适配和 key 更新。  
   - 尚未覆盖多指中断拖拽、重叠围栏候选列表、迷你标题条交互等关键验收行为。

## 建议修订方向

1. 将 40px 容差命中明确为表现层/地图投影层能力，基于屏幕坐标计算点到线段最短距离。  
2. 修正文档冲突：明确 `domain` 是否变更；若变更则删除“domain 层不变”表述。  
3. 统一手势规则文案，确保“目标-细则-验收”三处一致。  
4. 视觉规范改为主题 token，不使用硬编码颜色。  
5. 扩充测试清单并与验收项逐条映射，补充新增交互元素 key 约束。

## 处理建议

- 将文档状态维持为 `Draft`，完成以上问题修订后再进入开发实现。  
- 修订后建议进行一次快速复审，确认无规范冲突与验收缺口。

   - 文档要求新增 `AppColors.overlayDark`，但“文件变更”未列出 `core/theme/app_colors.dart`。

3. 重叠围栏候选项“牲畜数量”的数据来源未声明  
   - 文档要求候选项展示牲畜数量，但未说明数据来源（是否已在 `FenceState`，或需额外 provider）。

4. 更新头部 `范围` 描述，移除已删除文件引用。
