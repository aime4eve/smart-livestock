# b2b_admin UX 优化设计规格

> **文档编号**: SL-UX-2026-001
> **版本**: v1.0
> **编制日期**: 2026-05-03
> **状态**: 待实施
> **受众**: 产品经理 + 技术团队
> **前置文档**:
> - `docs/superpowers/specs/2026-05-02-unified-business-model-phase2b-design.md` (v1.2) — Phase 2b 规格
> - `docs/superpowers/plans/2026-05-02-unified-business-model-phase2b.md` — Phase 2b 实施计划（已完成）
> **范围**: b2b_admin 侧全部 4 个页面的 UX 升级

---

## 概述

Phase 2b 已交付 b2b_admin 侧边栏 5 项导航（概览/牧场/合同/对账/牧工管理），但 UI 体验存在以下共性问题：

1. **信息密度低** — 页面多为"标题 + 卡片列表"的扁平结构，缺少汇总指标和层级导航
2. **交互深度不足** — 无 drill-down 详情页，危险操作无二次确认，操作后无反馈
3. **数据展示粗糙** — 金额无单位格式化，日期无本地化，状态仅靠颜色区分
4. **视觉风格不统一** — 主卡片配色、图标使用（emoji vs Material Icons）不一致

本规格覆盖 b2b_admin 侧 4 个页面的完整 UX 升级，按优先级排序：概览页 > 对账 > 合同 > 牧工管理。

---

## 设计原则

- **Material Icons Outlined** — 全部图标使用 Material Icons Outlined 字体，禁止 emoji
- **低饱和灰蓝主色** — 主信息卡片使用 `linear-gradient(135deg, #37474f, #546e7a)`，与 AppColors.primary (#2E7D32) 搭配
- **层级导航** — 列表页 → 详情页（新路由），面包屑返回，不用 Dialog 承载核心内容
- **操作反馈** — 危险操作（确认对账/移除牧工/终止合同）必须有二次确认 Dialog + Loading + SnackBar 反馈
- **金额格式化** — `¥` 前缀 + 千分位逗号 + 两位小数（如 `¥5,460.00`）
- **日期本地化** — `2026年5月` 格式显示周期，`2026-01-15` 格式显示具体日期
- **状态语义化** — 绿/橙/红颜色 + 文字标签 + Material Icon 三重表达

### 1.2 主卡片设计

- 背景: `linear-gradient(135deg, #37474f, #546e7a)`
- 阴影: `0 4px 16px rgba(55,71,79,0.25)`
- 圆角: 14px
- 三列数据: 分润收入（从 revenue controller 获取本月数据）、总牲畜、待处理告警
- 底部进度条: 设备在线率（设备在线数/总数），背景 `rgba(255,255,255,0.2)`，前景 `#81c784`

### 1.3 告警提醒条

- 背景: `#fff3e0`
- 条件: 仅 `pendingAlerts > 0` 时显示
- 内容: 总告警数 + 最紧急牧场的告警摘要
- 点击跳转告警页（当前 b2b_admin 无独立告警页，跳转概览页即可）

### 1.4 快捷入口

4 列等宽网格，每格 `background: #f5f5f5`，圆角 10px，居中 Material Icon + 文字标签：
- 对账 → `AppRoute.b2bAdminRevenue`
- 合同 → `AppRoute.b2bAdminContract`
- 牧场 → `AppRoute.b2bAdminFarms`
- 牧工 → `AppRoute.b2bWorkerManagement`

### 1.5 牧场列表

每张牧场卡片包含：
- 牧场名称（`titleMedium` 加粗）
- 副标题：`280 头 · 12 设备 · 3 牧工`（用 `groups`/`pets`/`sensors` Material Icons）
- 右侧状态标签：正常（绿 `#e8f5e9`）/ 有告警（橙 `#fff3e0`，显示告警数）

### 1.6 数据来源

| 数据 | 来源 |
|------|------|
| 分润收入 | `revenueControllerProvider` 本月周期 |
| 总牲畜/总设备 | `b2bDashboardControllerProvider` 现有数据 |
| 待处理告警 | `b2bDashboardControllerProvider.pendingAlerts` |
| 牧场列表 | `b2bDashboardControllerProvider.farms` |

```
点击"确认对账"按钮
  → Dialog 二次确认（"确认本期对账数据无误？确认后不可撤回。"）
    → 按钮进入 Loading 状态
      → API 调用
        → 成功: SnackBar "对账已确认" + 状态标签更新
        → 失败: SnackBar 错误信息 + 按钮恢复可点击
```

### 2.5 空状态 / 异常

| 场景 | 处理 |
|------|------|
| 无周期 | "暂无对账数据，系统将在每月1日自动生成结算周期" |
| 加载失败 | 错误图标 + "加载失败" + 重试按钮 |
| 确认失败 | SnackBar 错误提示 + 按钮恢复 |

### 3.5 订阅服务状态区块

**显示条件**: 仅当 `tenant.billingModel === 'licensed'` 时显示。

- 外层: 同合同条款区样式
- 标题行: `vpn_key` icon + "订阅服务状态"
- 内层白底卡片：
  - 状态行：绿/黄/红圆点 + 状态文字 + Premium tier 渐变徽章
  - 4 格信息：`cloud` 部署方式 / `devices` 设备配额 / `favorite` 最近心跳 / `timer` 到期日

**状态展示规则**:

| 服务状态 | 圆点颜色 | 文字 | 补充信息 |

| active | 绿 `#4caf50`（带发光阴影） | 运行正常 | — |
| grace_period | 黄 `#ff9800` | 即将过期 | "心跳超时 N 小时" |
| degraded | 红 `#c62828` | 已降级 | "请续期恢复服务" |
| revoked/expired | 灰 `#999` | 已失效 | — |

### 3.6 快捷操作

### 3.7 空状态

### 4.4 操作流程

**分配牧工:**

```
点击"分配牧工"按钮
  → Dialog 弹出可用牧工列表（不在该牧场的工人）
    → 多选 Checkbox
      → 确认分配
        → 成功: SnackBar "已分配 N 名牧工到XX牧场" + 列表刷新
        → 失败: SnackBar 错误信息
```

**移除牧工:**

```
点击"移除"按钮
  → Dialog 二次确认（"确认将 XXX 从XX牧场移除？"）
    → 确认
      → 成功: SnackBar "已将 XXX 从XX牧场移除" + 列表刷新
      → 失败: SnackBar 错误信息
```

**批量分配:**

```
点击"批量分配牧工"入口
  → Dialog 选择目标牧场（下拉）
    → 选择牧工（多选）
      → 确认分配
        → 成功: SnackBar 反馈 + 列表页刷新
```

## 5. 共用交互规范

### 5.1 二次确认 Dialog

危险操作（确认对账/移除牧工/终止合同）统一使用 AlertDialog：
- 标题: 操作描述（"确认本期对账数据无误？"）
- 副标题: 补充说明（"确认后不可撤回。"）
- 按钮: "取消"（TextButton）+ "确认"（红色/主色 FilledButton）

### 5.2 SnackBar 反馈

操作成功/失败后显示 SnackBar：
- 成功: `ScaffoldMessenger.showSnackBar`，绿色 icon
- 失败: 红色 icon + 错误信息

### 5.3 Loading 状态

按钮 Loading 时:
- 按钮显示 `CircularProgressIndicator`（白色/主色，16px）
- 按钮禁用，文字替换为 "处理中..."
- API 完成后恢复
